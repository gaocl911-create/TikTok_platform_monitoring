# 阶段 4.5：真实内容采集与稳定性建设

## 阶段目标

阶段四已经完成抖音公开主页真实账号指标采集。阶段 4.5 将继续补齐真实作品发现与采集任务稳定性，
为评论监控、内容排行、爆款预警和日报周报提供可靠的数据基础。

## 当前基线

- `douyin_public_web` 可以通过标准无头 Edge 获取公开账号指标。
- 真实账号快照会记录采集器和数据质量。
- 真实采集失败不会回退到模拟数据。
- 抖音作品列表当前可能返回“服务异常”，系统会标记为 `partial`。
- Celery Worker 与 Beat 已可执行定时采集，但尚未实现同账号任务去重和自动重试。
- 已保存采集运行记录，但尚未提供运行记录列表 API 和前端页面。

## 当前实施进度（2026-06-04）

- 已实现严格的抖音账号作品区解析，只接受 `user-post-list` 内的作品链接。
- 已阻止热门推荐、页脚和其他非账号作品链接入库。
- 已支持真实作品字段不可用状态；未公开的发布时间和互动指标不会以 `0` 冒充真实值。
- 已实现 Redis 同账号采集锁、Celery 有限重试与指数退避。
- 手动采集已接入同一套 Redis 锁；临时渲染失败会返回排队状态并交给 Celery 自动重试。
- 账号表单与 API 可从抖音分享文案中提取纯短链，同时拒绝包含空格的普通主页地址。
- 抖音页面渲染超时已从 45 秒调整为 75 秒；超时后会强制终止本次 Edge 进程树。
- 已记录触发来源、尝试次数、采集器、错误类型和执行耗时。
- 已实现连续失败系统预警。
- 已提供采集运行记录列表、详情 API 和前端“采集运行”页面。
- 已对真实账号 `40877664675` 完成端到端验证；账号公开指标可采集，作品区本次仍返回服务异常，
  因此系统保持 `partial/unavailable`，不写入任何模拟作品或热门推荐。

当前尚未满足“至少一个真实账号成功发现公开作品”的验收项。原因是已测试账号的公开作品区仍由平台返回
服务异常；代码能力已就绪，待标准公开页面返回账号作品卡片后即可按严格归属规则保存。

## 实施范围

### 1. 真实作品采集

- 解析公开作品 ID、标题、链接、发布时间和封面。
- 解析页面公开展示的点赞、评论、收藏和分享指标。
- 识别新增作品并触发新内容预警。
- 重复采集只更新作品及快照，不重复创建作品。
- 作品列表不可用时保留真实账号指标，并明确记录内容采集状态。

### 2. 采集稳定性

- 使用 Redis 锁避免同一账号同时执行多个采集任务。
- 临时性浏览器渲染失败使用 Celery 自动重试与指数退避。
- 永久配置错误、账号不匹配和解析结构变化不进行无限重试。
- 将错误分类、采集器版本、执行耗时和重试次数写入采集运行记录。
- 连续失败达到阈值时生成系统预警。

### 3. 运行可观测性

- 提供采集运行记录列表和详情 API。
- 支持按账号、状态和数据来源筛选。
- 前端展示最近运行状态、执行耗时、结果摘要和失败原因。
- 运行记录页面清楚区分成功、部分成功、失败和跳过。

## 合规与真实性规则

- 只采集标准浏览器能够访问的公开页面数据。
- 不绕过登录验证、验证码、访问控制或平台安全措施。
- 不保存或复用个人登录态。
- 真实数据不可用时不生成模拟内容补全。
- 所有账号快照、作品和采集运行均保留数据来源信息。

## 实施顺序

```text
公开作品能力探测
→ 真实作品解析与测试
→ Celery 同账号任务去重
→ 有限重试与失败分类
→ 连续失败系统预警
→ 采集运行记录 API
→ 前端运行记录页面
→ 真实账号端到端验收
```

## 验收标准

- 至少一个真实抖音账号可以发现并保存公开作品。
- 重复采集不会重复创建同一作品。
- 同一账号不会被多个 Celery 任务同时执行。
- 临时失败可以自动重试，永久失败不会无限重试。
- 连续失败达到阈值时生成系统预警。
- 前端可以查看采集运行状态、耗时、摘要和失败原因。
- 真实作品列表不可用时，内容数量保持真实，不写入模拟作品。

## 暂不包含

- 绕过平台风控或验证码。
- 账号后台播放量、曝光量、完播率等非公开数据。
- 自动回复评论。
- 小红书真实作品采集。
