# 阶段四实施记录：抖音真实公开数据接入

## 完成范围

阶段四已接入首个真实数据适配器 `douyin_public_web`。它使用本机 Microsoft Edge
标准无头浏览器渲染抖音公开主页，并读取页面公开展示的账号指标。

当前可真实采集：

- 昵称、简介、IP 属地
- 关注数、粉丝数、累计获赞数、公开作品数
- 真实采集时间和账号指标历史快照

当前尚未稳定接入：

- 作品明细列表及单条作品互动指标
- 评论、后台播放量和其他仅账号本人可见的数据
- 小红书真实公开数据

## 数据真实性规则

系统不允许真实采集失败后静默回退到模拟数据。

账号支持两种采集器：

- `mock`：开发和演示数据，会生成模拟账号指标与模拟作品。
- `douyin_public_web`：抖音公开主页真实数据，不会用模拟内容补全缺失字段。

数据质量状态：

- `pending`：已选择真实采集，等待首次执行。
- `mock`：当前为模拟数据。
- `verified`：真实账号指标和内容明细均采集成功。
- `partial`：真实账号指标成功，但部分数据暂不可用。
- `failed`：真实采集失败，系统保留失败原因，不写入假数。

每条账号快照会保存 `collector_type` 和 `data_quality_status`。账号从模拟模式切换到真实模式时，
系统会清空当前模拟指标并删除该账号的模拟作品，避免假数据继续显示为当前数据。历史模拟快照仍会保留来源，
详情页趋势只展示当前数据来源的快照。

## 真实采集流程

```text
Celery 定时任务 / 手动立即采集
→ 根据 creator.collector_type 选择采集器
→ Edge 标准无头浏览器渲染公开主页
→ 验证主页抖音号与配置账号一致
→ 解析公开账号指标
→ 保存真实快照和来源状态
→ 作品列表不可用时标记 partial，不生成模拟作品
```

## 环境要求

Windows 开发环境默认使用：

```text
C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe
```

如 Edge 安装在其他位置，可在 `.env` 中配置：

```dotenv
DOUYIN_BROWSER_PATH=
DOUYIN_RENDER_TIMEOUT_SECONDS=45
DOUYIN_VIRTUAL_TIME_BUDGET_MS=15000
```

采集器不保存登录态，不绕过验证码或平台风控。当平台要求登录、验证码或人工验证时，采集会明确失败并记录原因。

## 使用方式

1. 在“监控账号”中添加或编辑抖音账号。
2. 数据来源选择“真实公开数据”。
3. 填写准确的抖音号和公开主页或分享链接。
4. 点击“添加并采集”或“立即采集”。
5. 在详情页查看数据来源、数据质量和采集说明。
